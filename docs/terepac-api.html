Welcome

The Terepac ONE RESTful API uses JSON Web Tokens (JWT) to authenticate each request. Each secured request requires the following 2 HTTP headers:

x-client-id: [client ID]
x-access-token: [jwt]

As we are only in alpha right now, only Terepac partners can get a login. If you are a partner, please contact Darryl Patterson. Please see the AUTHENTICATION documentation to find out how to get your JWT.

AUTHENTICATION

Pass on the email address and password as JSON. The system will return a JWT that will be used for future secured API requests.

If there is an issue with the client ID or JWT, the system will return an error as a JSON document. Here are the two possible error responses:

HTTP status code 401
{
  message: 'The supplied x-access-token (JWT) is not valid.'
}

HTTP status code 401
{
  message: 'One or both of the required headers (x-client-id, x-access-token) are missing.'
}

  • Response
  • Security

URI Parameters

  • id: required (string)

    The clients's ObjectId.

Query Parameters

  • tags: required (string)

    A comma delimited list of tags.

    Examples:

    tags=D001_LOC1_TI,D002_LOC1_EI
  • start: required (string)

    The start date of the data.

    Examples:

    start=2016-01-29T16:04:27.000Z
  • end: required (string)

    The end date of the data.

    Examples:

    end=2016-01-29T16:04:27.000Z
  • asset: (string)

    Include asset data.

    Examples:

    asset=1
  • device: (string)

    Include device data.

    Examples:

    device=1
  • sensor: (string)

    Include sensor data.

    Examples:

    sensor=1
